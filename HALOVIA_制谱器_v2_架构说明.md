# HALOVIA 制谱器 v2 — 架构与逻辑说明

> 本文档基于对 v2 全部源码（index.html + style.css + script.js）的完整分析撰写，
> 作为 v3R 开发的参考基础。

---

## 1. 项目定位

HALOVIA 制谱器是一款面向 **3D 无轨下落式键盘音乐游戏《HALOVIA》** 的 **交互式谱面编辑工具**。
以纯前端网页形式提供，不依赖任何外部框架或库，支持本地 `file://` 直接打开使用。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| 轨道路径编辑 | 通过分段函数（6 种预设 + 自定义表达式）定义判定环的移动轨迹 |
| 音符编排 | 在时间轴上添加/编辑/删除 Tap / Keep / Hold 三种音符 |
| 3D 实时预览 | Canvas 渲染 3D 透视下的音符下落、判定环运动、Hold 拖尾 |
| 试玩模式 | 键位映射后全屏游玩，含完整判定系统（P/G/B/M） |
| 导入/导出 | `.hvp`（紧凑十进制编码）和 `.hvf`（JSON 完整信息）双格式 |

## 4. 核心数据模型

### 4.1 全局状态（`state` 对象）

```js
const state = {
  bpm: 120,                      // 节拍/分钟
  sampleRateReal: 10.0,          // 采样频率 (Hz)
  offsetSec: 0,                  // 音频偏移 (秒)
  duration: 30,                  // 总时长 (秒)
  fov: 55,                       // 视场角 (度)

  tracks: [],                    // 轨道数组
  openedTracks: [],              // 当前打开的轨道 ID 列表
  notes: [],                     // 音符数组

  audioBuffer: null,             // AudioBuffer
  audioCtx: null,                // AudioContext
  audioSource: null,             // AudioBufferSourceNode
  audioStartedAt: 0,             // performance.now() 时间戳
  audioStartOffset: 0,           // 播放起始偏移

  keyMap: {},                    // {trackId: keyString}
};
```

### 4.2 轨道模型（Track）

```js
{
  id: 0,                         // 轨道编号
  segments: [                     // 路径分段数组
    {
      startTime: { raw: "0", value: 0 },      // 起始时间（表达式对象）
      endTime:   { raw: "30", value: 30 },    // 结束时间（表达式对象）
      preset: "static",                        // 预设名称
      params: {                                // 预设参数
        cx: { raw: "0", value: 0 },
        cy: { raw: "0", value: 0 }
      }
    }
  ],
  _sampledPath: null             // 导入 .hvp 后的采样路径（如有）
}
```

### 4.3 音符模型（Note）

```js
{
  track: 0,          // 所属轨道编号
  type: 1|2|3,        // 1=Tap, 2=Keep, 3=Hold
  time: 2.5,         // 命中时刻（秒）
  endTime: 3.0       // 仅 Hold 类型：尾部时刻（秒）
}
```

### 4.4 表达式对象（ExprObj）

```js
{
  raw: "pi/4",     // 用户输入的原始字符串
  value: 0.7854    // 解析后的数值
}
```

关键设计：**保留原始表达式字符串**，在 `.hvf` 中保存 `raw` 而非 `value`，
允许用户使用 `2*pi`、`sin(pi/4)` 等数学表达式作为参数值。

### 4.5 游戏运行时状态（`game` 对象）

```js
const game = {
  mode: "preview" | "play",   // 模式
  playing: true/false,         // 是否正在播放
  paused: true/false,          // 是否暂停
  gameTime: 0,                 // 当前游戏时间
  startRealTime: 0,            // 启动时的 performance.now()
  startGameTime: 0,            // 启动时的游戏时间
  combo: 0,                    // 连击数
  score: 0,                    // 得分（满分 1,000,000）
  judgments: {},               // {noteIdx: {judged, result, judgeTime, holdState}}
  effects: [],                 // 视觉效果数组
  keysDown: {},                // 当前按下的键 {keyStr: true}
  holdRingShrink: {},          // Hold 判定环收缩动画状态
  fullscreen: false,           // 是否全屏模式
};
```

## 5. 表达式系统

### 5.1 架构

整个制谱器的所有数字输入框都支持 **数学表达式**，核心机制：

| 函数 | 说明 |
|------|------|
| `evalNumericInput(str)` | 解析字符串为数值，支持 pi, e, sin, cos, sqrt 等 |
| `makeExpr(input)` | 构造 {raw, value} 表达式对象 |
| `toExpr(v, fallback)` | 将任意值规范化到表达式对象 |
| `exprVal(v, fallback)` | 取表达式的数值 |
| `exprRaw(v, fallback)` | 取表达式的原始字符串 |
| `bindExprInput(el, onCommit)` | 绑定输入框为表达式模式 |

### 5.2 支持的运算

- **常量**: pi, PI, e, E
- **运算符**: +, -, *, /, ^（幂）
- **三角函数**: sin, cos, tan, asin, acos, atan, atan2
- **数学函数**: sqrt, abs, pow, log, log2, log10, exp
- **取整**: floor, ceil, round
- **极值**: min, max

### 5.3 输入行为

- 输入实时解析，有效则去红框，无效则标红
- 不回写为数值，保留表达式字符串
- 双击输入框弹出放大编辑窗口（大号输入框 + 遮罩层）
- Enter 确认，Esc 取消，点击遮罩确认
- .hvf 导出时保留原始表达式

## 6. 轨道路径系统

### 6.1 分段机制

每个轨道的运动路径被划分为若干 **分段 (Segment)**，每段定义：
- 时间范围 [startTime, endTime]
- 运动预设 + 参数

在任意时刻 t，通过 evalTrackAt(track, t) 计算轨道位置：

1. 查找包含 t 的分段 → 计算归一化进度 p = (t - start) / duration
2. 用预设函数计算 (x, y) 坐标
3. 若超出所有分段范围 → 取最后一个分段终点值（保持静止）
4. 若在第一个分段之前 → 取第一个分段起点值

### 6.2 预设运动类型

| 预设 | 参数 | 公式 | 用途 |
|------|------|------|------|
| hidden | 无 | 无 | 隐藏轨道 |
| static | cx, cy | 常数坐标 | 固定位置判定环 |
| line | x0,y0, x1,y1 | 线性插值 | 直线移动 |
| ellipse | cx,cy, ax,ay, omega, phiX,phiY | 三角函数 | 圆形/椭圆路径 |
| lissajous | cx,cy, ax,ay, omegaX,omegaY, phiX,phiY, funcX,funcY | 双轴独立三角函数 | 李萨如曲线 |
| custom | xExpr, yExpr | JavaScript 表达式求值 | 任意路径 |

自定义表达式中可以使用变量：
- p = 分段内归一化进度 [0, 1]
- P = 全局归一化进度 [0, 1]
- t = 分段内经过的时间
- T = 全局时间

### 6.3 校验系统

| 校验项 | 级别 | 说明 |
|--------|------|------|
| 分段结束 ≤ 起始 | ❌ 错误 | 时间顺序无效 |
| 分段时间重叠 | ❌ 错误 | 同一轨道两段不能重叠 |
| 坐标超 [-2, 2] | ❌ 错误 | 路径越界 |
| 隐藏段内有音符 | ❌ 错误 | 旋律丢失 |
| BPM/采样率/偏移超范围 | ❌ 错误 | 编码容量超限 |
| 音符时间戳超 5 位 | ❌ 错误 | 超过 99999 |
| 相邻分段不连续 | ⚠️ 警告 | 坐标跳变 |

校验结果显示在每个分段卡片内部（红/黄色背景）。

## 7. 3D 预览系统

### 7.1 透视投影

```
视点位于 Z = -viewDist（viewDist = 1/tan(FOV/2)）
3D 点 (nx, ny, z) → 屏幕坐标：
  dz = z + viewDist
  sx = cx + nx × viewDist/dz × scale
  sy = cy - ny × viewDist/dz × scale
```

### 7.2 音符显示

| 类型 | 外观 | 说明 |
|------|------|------|
| Tap | 圆环（空心） | 外圈 + 黑色内圈 |
| Keep | 光团 + 细环 | 径向渐变中心光 + 外圈细环 |
| Hold | 实心圆 | 填充圆 + 描边 |

### 7.3 Hold 拖尾

Hold 音符的中间体用 **多边形路径**（左右两侧点序列）渲染为 3D 圆柱体：
- 在 [headTime, tailTime] 之间等距采样 20 个点
- 每个点取对应时刻的轨道位置 (x, y) 和 Z 深度
- 投影到屏幕后构造封闭多边形
- 多押时附加白色外发光

### 7.4 判定特效

- **Perfect**: 轨道色扩散环（0.2 秒淡出）
- **Good**: 白色扩散环
- 多押音符有白色外发光标记

### 7.5 节拍线

按 BPM 计算节拍时长 beatDur = 60/BPM，在当前可见 Z 范围内绘制水平节拍线。

## 8. 判定系统

### 8.1 判定窗口

| 等级 | 误差范围 | 得分倍率 |
|------|----------|----------|
| Perfect | -60ms ~ +60ms | ×1.0 |
| Good | -100ms ~ -60ms, +60ms ~ +100ms | ×0.6 |
| Bad | -120ms ~ -100ms, +100ms ~ +120ms | ×0 |
| Miss | > +120ms | ×0 |
| 无效 | < -120ms | 忽略 |

### 8.2 得分公式

```
每音符分值 = 1,000,000 / 总音符数
最终得分 = ∑(每音符分值 × 倍率)
```

### 8.3 音符类型差异

| 类型 | 判定逻辑 |
|------|----------|
| Tap | 按下瞬间判定，无需保持 |
| Keep | 按住且保持覆盖命中瞬间的 ±60ms |
| Hold | 头部=Tap 判定，尾部=需保持按压直到尾部前 60ms |

### 8.4 Hold 尾部特殊规则

- 头部判定成功 (P/G) 后进入 holding 状态
- 判定环缩小 20%（视觉反馈）
- 尾部到达前若松手 → broken → 撤销得分，计为 Miss
- 尾部到达后 → done → 判定环恢复

## 9. 导入/导出格式

### 9.1 .hvp — 紧凑十进制编码

单行数字字符串，编码结构：

```
BPM(3) 采样频率(3) 时间偏移周期(3) | 轨道数(2) | 轨道长度N | 轨道数据(8×N) | 音符数M(8) | 音符数据(8×M)

轨道数据: X坐标(4) Y坐标(4)
音符数据: 轨道(2) 类型(1) 时间戳(5)
```

- 所有字段为十进制数字，按位宽拼接
- 采样频率存储值 = round(实际Hz × 10)
- 偏移存储值 = round(偏移秒 × 实际Hz) + 500
- 时间戳 = round(秒数 × 实际Hz)
- 轨道坐标 存储值 = round(值 × 1000 + 5000)

编码格式的每位数据范围决定了限制：
- BPM: 3 位 → 0~999
- 轨道: 2 位 → 最多 99 条（实际限 80）
- 音符时间戳: 5 位 → 0~99999

### 9.2 .hvf — JSON 完整格式

```json
{
  "version": "1.0",
  "meta": { "bpm": 120, "sampleRateReal": 10, ... },
  "tracks": [
    {
      "id": 0,
      "segments": [...]
    }
  ],
  "notes": [...]
}
```

保留完整的表达式原始字符串，可 **完整恢复编辑状态**。

## 10. 时间轴系统

### 10.1 渲染

- Canvas 绘制，包含：时间刻度尺、轨道水平条、音符图形、播放游标
- 每条轨道对应一个水平条，颜色与轨道色相一致
- 时间自适应刻度（60s → 30s → 10s → 5s → 2s → 1s → 0.5s → ...）
- 节拍线：主节拍 + 次节拍分割（1/2, 1/3, 1/4, 1/6, ... 自适应）
- 吸附线：采样周期整倍数的淡蓝色竖线

### 10.2 交互

| 鼠标操作 | 效果 |
|----------|------|
| 轨道内左键单击 | 添加 Tap 音符（吸附到采样周期） |
| 轨道内右键单击 | 添加 Keep 音符 |
| 右键拖动 (>5px) | 添加 Hold 音符（起点→终点） |
| 双击音符 | 打开编辑弹窗 |
| 按住音符拖动 | 平移音符（可跨轨道） |
| Hold 右端拖动 | 调整持续时长 |
| 标尺区单击 | 跳转播放位置 |
| 滚轮 | 水平平移 |
| Ctrl + 滚轮 | 以鼠标为中心缩放 |
| 空白拖动 | 水平平移 |

### 10.3 吸附系统

- 吸附到 **采样周期的整倍数**
- 倍数 k 由缩放级别自适应选择（优先"友好"值：1,2,3,4,5,6,8,10,12...）
- 可见区域保持约 20 个吸附位点
- Alt 键临时禁用吸附

## 11. 试玩系统

### 11.1 键位映射

- 弹窗列出所有轨道
- 点击轨道行 → 按键盘按键绑定（支持 Ctrl/Shift/Alt 修饰键）
- 4~8K 有默认键位方案
- 绑定记忆存储于 localStorage（key = halovia_keymap_轨道列表）
- 所有轨道必须绑定后才能开始

### 11.2 游戏流程

1. 确认键位 → 进入全屏模式（伪全屏，CSS position: fixed）
2. 左上角显示"暂停"按钮
3. 播放音频 + 启动游戏循环（requestAnimationFrame）
4. 按键判定：预览模式→自动Perfect；试玩模式→根据按键时机和判定窗口结算
5. 暂停弹窗：继续 / 重试 / 退出
6. 播放结束后显示结算分数（满分 1,000,000）
7. Esc 直接退出试玩

### 11.3 音频同步

- 使用 Web Audio API (AudioContext.decodeAudioData)
- 播放时记录 audioStartedAt，游戏时间 = audioStartOffset + (now - audioStartedAt)
- 暂停/恢复时处理时间偏移

### 12.1 文件加载

- 打击音效文件存放于 sound/ 目录
- 自动尝试 .mp3 → .ogg → .m4a 扩展名
- 每种类型预加载 8 个 audio 元素组成池

### 12.2 音量控制

- 竖向滑块（CSS writing-mode: vertical-lr）
- 音效音量 / 音乐音量独立控制
- 音量值保存到 localStorage（halovia_vol_hit, halovia_vol_music）

### 12.3 已知限制

- file:// 协议下 audio 元素有 20~100ms 延迟
- file:// 下控制台会有文件探测报错（尝试多种扩展名导致）

## 15. 关键设计决策

| 决策 | 说明 |
|------|------|
| 纯原生无依赖 | 零外部库，支持 file:// 离线使用 |
| 表达式输入 | 所有数字字段支持数学表达式，保留原始字符串 |
| 分段路径 | 轨道路径由时间分段 + 预设函数组成，灵活且可还原 |
