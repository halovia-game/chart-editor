# 制谱器开发后检查清单

每次完成功能开发后，逐项检查以下要点，避免遗漏。

## 1. UI 同步

- [ ] 状态修改后调用 `renderTrackPanels()` 同步左侧面板
- [ ] 状态修改后调用 `renderTimeline()` 同步时间轴
- [ ] 状态修改后调用 `scheduleSizeEstimate()` 更新编码预估
- [ ] 音频时长变化后更新 `state.duration` 和全局参数输入框
- [ ] 追加/删除关键帧后调用 `renderTrackPanels()`（新增末尾关键帧尤其容易漏）
- [ ] 关键帧拖拽过程中实时调用 `renderTrackPanels()`（预览区拖拽 + 时间轴拖拽）

## 2. 撤销系统

- [ ] `pushUndo()` 在状态**修改之前**调用，不是在之后
- [ ] 修改了 `state` 对象的新字段也要加入 `saveStateSnap()` 快照
- [ ] 撤销时 `applySnap()` 恢复所有字段
- [ ] 新操作要清空 `redoStack`
- [ ] `undo()` 和 `redo()` 用 `applySnap()` 而不是手写恢复逻辑

## 3. 数据一致性

- [ ] `applyStateToUI()` 和 `syncGlobals()` 双向同步所有全局参数
- [ ] `syncGlobals()` 中读 DOM → 写 `state`
- [ ] `applyStateToUI()` 中读 `state` → 写 DOM
- [ ] 新增的 `state` 字段在 `exportHVF` / `importHVF` 中都要处理
- [ ] `.hvf version` 号变更时考虑旧版兼容
- [ ] 关键帧数组始终保持时间有序

## 4. 音视频

- [ ] 音频加载后更新 `audioEnergy`（含 low/high 频率数组）
- [ ] 音频时长变化后确保各轨道末尾有关键帧
- [ ] 频谱/波形绘制考虑缩放级别自适应
- [ ] 倍速变化时同步 `gameTime` 连续性
- [ ] `loadAudioFromPath` 与 `handleAudioUpload` 行为一致

## 5. 交互

- [ ] 新增的按键快捷键加入帮助面板
- [ ] 鼠标操作不要与已有操作冲突（尤其是 Ctrl 键）
- [ ] 拖拽交互在 `mousedown` 设标志、`mousemove` 执行、`mouseup` 清标志
- [ ] 关键帧拖拽用 Ctrl+左键，不与音符拖拽冲突
- [ ] 弹窗点击遮罩外部可以关闭（`onclick="if..."`）

## 6. 渲染

- [ ] Canvas 绘制用 `fill`+`stroke` 同路径避免缺口
- [ ] 时间轴元素大小随 `trackH` 缩放
- [ ] 缩放/平移时能量波形连续不跳变
- [ ] 能量波形覆盖 [viewStart, viewEnd] 全区间，不留空白
- [ ] 绘制顺序：背景 → 能量 → 刻度 → 轨道线 → 关键帧 → 音符 → 拖拽中的关键帧（置顶）→ 红色游标
- [ ] `ctx.beginPath()` 不要在循环外残留

## 7. 边界条件

- [ ] 轨道数为 0 时各渲染函数不崩溃
- [ ] 关键帧数小于 2 时 `evalTrackAt` 返回默认值
- [ ] 音符超出 `state.duration` 时截断或拒绝
- [ ] 滑块值在 `Math.max`/`Math.min` 范围内
- [ ] 音频未加载时相关函数安全降级
- [ ] `.hvp` 帧数字段 5 位够不够？（当前最大 99999）

## 8. 性能

- [ ] 能量波形每帧不超过 ~200 个 path 点
- [ ] `renderTimeline` 不在每帧做 DOM 查询（缓存 `getElementById`）
- [ ] `renderTrackPanels` 不在拖拽每帧调用（80ms 节流）
- [ ] 大数组遍历（FFT、RMS）只在音频加载时做一次
